## Introdução
<p align = "justify">
O brainstorm é uma técnica de elicitação de requisitos que consiste em reunir a equipe para discutir ideias sobre o projeto. Neste contexto, foi utilizado para levantar funcionalidades e características de uma solução voltada à gestão de atividades acadêmicas complementares, incentivando a participação de todos sem julgamentos.
</p>

## Metodologia
<p align = "justify">
A equipe se reuniu de forma remota para debater ideias sobre o sistema de atividades acadêmicas complementares. A sessão teve início com perguntas norteadoras e contou com um moderador responsável por conduzir a discussão e registrar as contribuições dos participantes.
</p>

## Versão 1.0

## Perguntas

### 1. Qual o objetivo principal da aplicação?

<p align = "justify">
<b>Aluno 1</b> - Deve ser uma plataforma onde os estudantes possam registrar suas atividades complementares.

<b>Aluno 2</b> - A plataforma deve permitir acompanhar a carga horária exigida pelo curso.

<b>Aluno 3</b> - O objetivo é facilitar a organização das atividades realizadas ao longo da graduação.

<b>Aluno 4</b> - O sistema deve auxiliar na validação das atividades pela coordenação.

<b>Aluno 5</b> - A plataforma deve centralizar todas as informações acadêmicas complementares.
</p>

---

### 2. Como será o processo para cadastrar um novo usuário?

<p align = "justify">
<b>Aluno 1</b> - O usuário deverá criar uma conta com dados pessoais e acadêmicos.

<b>Aluno 2</b> - O sistema pode integrar com o login institucional.

<b>Aluno 3</b> - Após o cadastro, o usuário poderá acessar suas atividades.

<b>Aluno 4</b> - Deve haver validação de e-mail para segurança.

<b>Aluno 5</b> - O perfil deve incluir curso e período.
</p>

---

### 3. Como será a forma de adicionar atividades?

<p align = "justify">
<b>Aluno 1</b> - O aluno poderá cadastrar atividades manualmente.

<b>Aluno 2</b> - Deve ser possível anexar certificados.

<b>Aluno 3</b> - As atividades devem ter categorias (ensino, pesquisa, extensão).

<b>Aluno 4</b> - O sistema deve permitir editar ou excluir atividades.
</p>

---

### 4. Outras perguntas pertinentes ao contexto

<p align = "justify">
<b>Aluno 1</b> - O sistema pode enviar notificações sobre prazos.

<b>Aluno 2</b> - Deve haver um painel com progresso da carga horária.

<b>Aluno 3</b> - A coordenação pode validar ou recusar atividades.

<b>Aluno 4</b> - Relatórios podem ser gerados automaticamente.
</p>

---

### 5. Como será a validação das atividades?

<p align = "justify">
<b>Aluno 1</b> - A coordenação irá analisar os certificados enviados.

<b>Aluno 2</b> - O sistema pode indicar atividades pendentes.

<b>Aluno 3</b> - Deve haver feedback em caso de rejeição.
</p>

---

### 6. Quais informações seriam interessantes para o usuário?

<p align = "justify">
<b>Aluno 1</b> - Total de horas concluídas.

<b>Aluno 2</b> - Atividades pendentes e aprovadas.

<b>Aluno 3</b> - Histórico completo das atividades.

<b>Aluno 4</b> - Regras e diretrizes da instituição.
</p>

## Conclusão
<p align = "justify">
A aplicação da técnica de brainstorm permitiu levantar ideias importantes e definir requisitos iniciais para o desenvolvimento de um sistema de gestão de atividades acadêmicas complementares.
</p>
